1Ô∏è‚É£ Variables: let, const, var

‚ùå Old Way (var)
var name = "John"; var name = "Jane"; // Can redeclare! name = "Bob"; // Can change

‚ö†Ô∏è Avoid var - it's old and confusing!

‚úÖ Modern Way
const name = "John"; // Can't change let age = 25; // Can change age = 26; // ‚úÖ Works! // name = "Jane"; ‚ùå Error!

‚ú® Use const by default, let when needed

üí° Key Rules:

  • const - Use for values that don't change (default choice!)
  • let - Use when value needs to change (counters, toggles)
  • var - Don't use! It's outdated

‚öõÔ∏è React Connection:

In React, you'll use const for most things - components, hooks, props. Use let rarely!

2Ô∏è‚É£ Arrow Functions

‚ùå Old Function Syntax
function greet(name) { return "Hello " + name; } const result = greet("John");
‚úÖ Arrow Function (Modern)
const greet = (name) => { return `Hello ${name}`; }; // Even shorter (implicit return): const greet = (name) => `Hello ${name}`;

Different Arrow Function Styles:

// No parameters const sayHi = () => "Hi!"; // One parameter (parentheses optional) const double = num => num * 2; // Multiple parameters const add = (a, b) => a + b; // Multiple lines (need curly braces and return) const calculate = (a, b) => { const sum = a + b; return sum * 2; };

üéÆ Try It: Arrow Functions

‚öõÔ∏è React Connection:

React uses arrow functions EVERYWHERE - event handlers, array methods, functional components. Master this!

3Ô∏è‚É£ Template Literals (Backticks)

‚ùå Old String Concatenation
const name = "John"; const age = 25; const message = "Hello " + name + ", you are " + age + " years old";

üò´ Messy and hard to read!

‚úÖ Template Literals
const name = "John"; const age = 25; const message = `Hello ${name}, you are ${age} years old`;

‚ú® Clean and readable!

üí° Template Literal Features:

  • Use backticks ` instead of quotes " or '
  • Embed variables with ${variable}
  • Can include expressions: ${2 + 2}
  • Multi-line strings work naturally!
// Examples const user = "Alice"; const score = 95; console.log(`${user} scored ${score}/100`); console.log(`Result: ${score > 90 ? "A+" : "Keep trying"}`);

‚öõÔ∏è React Connection:

You'll use template literals for dynamic class names, messages, and displaying data in JSX!

4Ô∏è‚É£ Destructuring - Unpack Values

üì¶ Object Destructuring

‚ùå Without Destructuring
const user = { name: "John", age: 25, city: "NYC" }; const name = user.name; const age = user.age; const city = user.city;
‚úÖ With Destructuring
const user = { name: "John", age: 25, city: "NYC" }; const { name, age, city } = user; // Done in ONE line! ‚ú®

üìö Array Destructuring

// Array destructuring const colors = ["red", "green", "blue"]; const [first, second, third] = colors; console.log(first); // "red" console.log(second); // "green" // Skip values const [, , thirdColor] = colors; console.log(thirdColor); // "blue"

‚öõÔ∏è React Connection:

SUPER IMPORTANT! React's useState hook uses array destructuring:

const [count, setCount] = useState(0); // This is destructuring! ‚òùÔ∏è

5Ô∏è‚É£ Array Methods: map, filter, find

üó∫Ô∏è map() - Transform Every Item

const numbers = [1, 2, 3, 4, 5]; // Double every number const doubled = numbers.map(num => num * 2); // Result: [2, 4, 6, 8, 10] // Convert names to uppercase const names = ["alice", "bob", "charlie"]; const upper = names.map(name => name.toUpperCase()); // Result: ["ALICE", "BOB", "CHARLIE"]

üîç filter() - Keep Only Some Items

const numbers = [1, 2, 3, 4, 5, 6]; // Keep only even numbers const evens = numbers.filter(num => num % 2 === 0); // Result: [2, 4, 6] // Keep only long names const names = ["Al", "Alexander", "Bo", "Benjamin"]; const longNames = names.filter(name => name.length > 5); // Result: ["Alexander", "Benjamin"]

üéØ find() - Get First Match

const users = [ { id: 1, name: "Alice" }, { id: 2, name: "Bob" }, { id: 3, name: "Charlie" } ]; const user = users.find(u => u.id === 2); // Result: { id: 2, name: "Bob" }

‚öõÔ∏è React Connection:

THIS IS CRUCIAL! You'll use map() constantly in React to render lists:

6Ô∏è‚É£ Spread Operator (...)

üìã Copy and Merge Arrays

const arr1 = [1, 2, 3]; const arr2 = [4, 5, 6]; // Copy array const copy = [...arr1]; // Merge arrays const merged = [...arr1, ...arr2]; // Result: [1, 2, 3, 4, 5, 6] // Add items const extended = [...arr1, 99, 100]; // Result: [1, 2, 3, 99, 100]

üì¶ Copy and Merge Objects

const person = { name: "John", age: 25 }; // Copy object const copy = { ...person }; // Add/Update properties const updated = { ...person, age: 26, city: "NYC" }; // Result: { name: "John", age: 26, city: "NYC" } // Merge objects const obj1 = { a: 1, b: 2 }; const obj2 = { c: 3, d: 4 }; const merged = { ...obj1, ...obj2 }; // Result: { a: 1, b: 2, c: 3, d: 4 }

üí° Why Use Spread?

  • Creates copies instead of references
  • Keeps original data unchanged (immutable)
  • Clean syntax for merging
  • Essential for React state management!

‚öõÔ∏è React Connection:

CRITICAL FOR REACT! You'll use spread to update state immutably:

// Updating state in React setUser({ ...user, age: 26 }); // Adding to array state setItems([...items, newItem]);

üéÅ BONUS: Ternary Operator (Short if-else)

‚ùå Long if-else
const age = 20; let status; if (age >= 18) { status = "Adult"; } else { status = "Minor"; }
‚úÖ Ternary Operator
const age = 20; const status = age >= 18 ? "Adult" : "Minor"; // condition ? ifTrue : ifFalse
// More examples const score = 85; const grade = score >= 90 ? "A" : score >= 80 ? "B" : "C"; const isLoggedIn = true; const message = isLoggedIn ? "Welcome back!" : "Please login";

‚öõÔ∏è React Connection:

Ternary operators are used EVERYWHERE in React for conditional rendering:

{isLoggedIn ? <Dashboard /> : <Login />}
