Understanding CSS Cascading

What is Cascading?
Sometimes multiple CSS rules try to style the same element. CSS uses "cascading" to decide which rule should actually apply. It's like a priority system!

Demo 1: Source Order (Last Rule Wins)

In the CSS, I wrote two rules for this text - one says green, another says orange. The orange rule comes second, so it wins!

When rules are equal, the last one in your CSS file wins.

Demo 2: Specificity Hierarchy

I'm styled by tag name 'p' - this is the weakest selector (blue text).

I have a class '.highlight' - classes are stronger than tags (purple and bold).

I have an ID '#special' - IDs are the strongest (red and underlined).

I have BOTH class and ID. The ID color (red) wins, but I still get bold from the class!

Power ranking: ID beats Class beats Tag

Demo 3: Combining Selectors

I'm inside a .box div, styled by '.box p' (brown).

I have class .text AND I'm inside .box, styled by '.box .text' (teal). Two classes beat one class + tag!

More specific selectors have higher priority.

Demo 4: The !important Declaration

I have two rules - one says pink, another says navy with !important. The !important keyword forces navy to win, ignoring all normal rules!

!important is like a trump card - use it carefully!

Demo 5: Inline Styles (Highest Priority)

Even though this paragraph has class and ID selectors that say red and purple, the inline style="color: green" wins! Inline styles beat everything (except !important).

Inline styles always win (unless !important is used).

Summary: The Cascade Priority Order

  1. Inline styles - style="..." (Highest Priority)
  2. ID selectors - #id
  3. Classes, attributes, pseudo-classes - .class, [attribute], :hover
  4. Elements & pseudo-elements - div, p, ::before
  5. Universal selector - * (Lowest Priority)

Special case: !important breaks all these rules and takes top priority!

Tie-breaker: When specificity is equal, the rule that comes LAST wins.

Remember: Cascading is like a waterfall - styles flow down, and more specific rules override general ones.
