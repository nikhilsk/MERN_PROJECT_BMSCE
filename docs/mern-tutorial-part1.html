üß© React Components

What are Components?

Components are reusable pieces of UI. Think of them like LEGO blocks - you build your app by combining them.

üí° Key Point: Components are JavaScript functions that return JSX (HTML-like syntax).

Your Three Main Components:

1. Users.jsx (List Component)

function Users() {
  const [users, setUsers] = useState([]);
  
  useEffect(() => {
    axios.get("http://localhost:3001")
      .then(result => setUsers(result.data))
  }, []);
  
  return (
    <div>
      <table>
        {users.map(user => (
          <tr key={user._id}>
            <td>{user.name}</td>
            <td>{user.email}</td>
          </tr>
        ))}
      </table>
    </div>
  )
}

export default Users;

What this component does:

  • üìä Displays a table of all users
  • üîÑ Fetches data from server on load
  • ‚úèÔ∏è Has Update and Delete buttons
  • ‚ûï Has "Add +" link to create new user

2. CreateUser.jsx (Form Component)

function CreateUser() {
  const [name, setName] = useState();
  const [email, setEmail] = useState();
  const [age, setAge] = useState();
  const navigate = useNavigate();

  const Submit = (e) => {
    e.preventDefault();
    axios.post("http://localhost:3001/createuser", {name, email, age})
      .then(() => navigate('/'))
  }

  return (
    <form onSubmit={Submit}>
      <input type="text" onChange={(e) => setName(e.target.value)} />
      <input type="email" onChange={(e) => setEmail(e.target.value)} />
      <input type="number" onChange={(e) => setAge(e.target.value)} />
      <button>Submit</button>
    </form>
  )
}

What this component does:

  • üìù Shows form with 3 input fields
  • üíæ Saves state for each input
  • üì§ Sends data to server on submit
  • ‚Ü©Ô∏è Redirects to home after success

3. UpdateUser.jsx (Edit Form)

function UpdateUser() {
  const {id} = useParams();  // Get ID from URL
  const [name, setName] = useState();
  
  useEffect(() => {
    // Fetch existing user data
    axios.get('http://localhost:3001/getuser/' + id)
      .then(result => setName(result.data.name))
  }, [id]);

  const Update = (e) => {
    e.preventDefault();
    axios.put("http://localhost:3001/updateuser/" + id, {name, email, age})
      .then(() => navigate('/'))
  }

  return (
    <form onSubmit={Update}>
      <input type="text" value={name} onChange={(e) => setName(e.target.value)} />
      <button>Update</button>
    </form>
  )
}

What this component does:

  • üîç Reads user ID from URL
  • üì• Fetches existing user data
  • üìù Pre-fills form with current values
  • üíæ Updates user in database
üéØ Component Best Practices:
  • One component = one responsibility
  • Keep components small and focused
  • Reuse components when possible
  • Export with export default ComponentName

üé£ React Hooks

Hooks are special functions that let you "hook into" React features in functional components.

‚ö†Ô∏è Hook Rules:
  • Only call hooks at the top level (not inside loops/conditions)
  • Only call hooks in React functions
  • Hook names must start with "use"

1. useState - Managing State

üí° What is State?
State is data that can change over time. When state changes, React re-renders the component to show the new data.

Basic Syntax:

const [value, setValue] = useState(initialValue);

// Example:
const [name, setName] = useState("");  // Empty string initially
const [age, setAge] = useState(0);     // Number
const [users, setUsers] = useState([]); // Empty array

Real Example from CreateUser.jsx:

const [name, setName] = useState();

// When user types in input:
<input 
  type="text" 
  onChange={(e) => setName(e.target.value)}
/>

// name updates on every keystroke
// Component re-renders with new value

How It Works:

Part Description Example
name Current value "John"
setName Function to update value setName("Jane")
useState("") Initial value Empty string

2. useEffect - Side Effects

üí° What are Side Effects?
Operations that reach outside your component: fetching data, timers, subscriptions, DOM manipulation.

Basic Syntax:

useEffect(() => {
  // Code runs after render
  
  return () => {
    // Cleanup (optional)
  }
}, [dependencies]);

Example from Users.jsx:

useEffect(() => {
  axios.get("http://localhost:3001")
    .then(result => setUsers(result.data))
    .catch(err => console.log(err));
}, []);  // Empty array = run once on mount

What happens:

  1. Component renders (empty table)
  2. useEffect runs after render
  3. Fetches data from server
  4. Updates state with data
  5. Component re-renders with data

Dependency Array Explained:

Dependency When It Runs Use Case
[] Once on mount Fetch initial data
[id] When id changes Fetch user by ID
No array Every render Rarely needed (can cause bugs)

Example with Dependency (UpdateUser.jsx):

useEffect(() => {
  axios.get('http://localhost:3001/getuser/' + id)
    .then(result => {
      setName(result.data.name);
      setEmail(result.data.email);
      setAge(result.data.age);
    })
}, [id]);  // Re-runs when 'id' changes

3. useParams - Get URL Parameters

import { useParams } from 'react-router-dom';

function UpdateUser() {
  const {id} = useParams();  
  // If URL is /update/abc123, id = "abc123"
  
  console.log(id);  // "abc123"
}

4. useNavigate - Programmatic Navigation

import { useNavigate } from 'react-router-dom';

function CreateUser() {
  const navigate = useNavigate();
  
  const Submit = (e) => {
    e.preventDefault();
    axios.post(url, data)
      .then(() => navigate('/'));  // Go to home
  }
}
üéØ When to Use Each Hook:
  • useState: Form inputs, toggles, counters
  • useEffect: API calls, subscriptions, timers
  • useParams: Read dynamic URL segments
  • useNavigate: Redirect after form submit

üì¶ Props & Event Handling

What are Props?

Props (properties) are arguments passed to components - like function parameters.

üí° Key Concept: Props flow one way - from parent to child (like a waterfall).

Basic Example:

// Parent Component
function App() {
  return <UserCard name="John" age={25} />
}

// Child Component
function UserCard(props) {
  return (
    <div>
      <h2>{props.name}</h2>
      <p>Age: {props.age}</p>
    </div>
  )
}

// Or with destructuring:
function UserCard({ name, age }) {
  return <div>{name} - {age}</div>
}

Props in Your Project (App.jsx):

<Route path='/' element={<Users />} />
<Route path='/create' element={<CreateUsers />} />

// 'element' is a prop passed to Route component

Event Handling

1. onChange - Input Changes

<input 
  type="text" 
  onChange={(e) => setName(e.target.value)}
/>

// e = event object
// e.target = the input element
// e.target.value = current input value

What Happens:

  1. User types "J" ‚Üí onChange fires
  2. setName("J") updates state
  3. Component re-renders
  4. User types "o" ‚Üí onChange fires again
  5. setName("Jo")
  6. Process repeats for each keystroke

2. onSubmit - Form Submission

const Submit = (e) => {
  e.preventDefault();  // Stop page reload!
  
  axios.post(url, {name, email, age})
    .then(result => console.log(result))
}

<form onSubmit={Submit}>
  <button type="submit">Submit</button>
</form>
‚ö†Ô∏è Always use e.preventDefault():
Without it, form submission reloads the page and loses all your React state!

3. onClick - Button Clicks

<button onClick={() => handleDelete(user._id)}>
  Delete
</button>

// Arrow function needed to pass arguments
// Without arrow: onClick={handleDelete(user._id)} 
// ‚ùå This would run immediately on render!

Delete Handler Example:

const handleDelete = (id) => {
  axios.delete(`http://localhost:3001/deleteuser/${id}`)
    .then(res => {
      console.log(res);
      // Update state to remove deleted user
      setUsers(users.filter(user => user._id !== id));
    })
}

üõ£Ô∏è React Router - Navigation

üí° Why React Router?
Allows navigation between pages without full page reloads - creating a smooth Single Page Application (SPA).

Setup in App.jsx

import {BrowserRouter, Routes, Route} from 'react-router-dom';
import Users from './Users';
import CreateUsers from './CreateUser';
import UpdateUsers from './UpdateUser';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path='/' element={<Users />} />
        <Route path='/create' element={<CreateUsers />} />
        <Route path='/update/:id' element={<UpdateUsers />} />
      </Routes>
    </BrowserRouter>
  )
}

Core Components

1. BrowserRouter

Purpose: Wraps your entire app to enable routing

Usage: Put at the top level (usually in App.jsx or main.jsx)

<BrowserRouter>
  {/* All your routes go here */}
</BrowserRouter>

2. Routes

Purpose: Container for all your Route components

Usage: Looks through all routes and renders the first match

3. Route

Purpose: Defines a path and what component to show

<Route path='/about' element={<About />} />

Route Types

URL Route Path Component Type
/ path='/' <Users /> Static
/create path='/create' <CreateUsers /> Static
/update/abc123 path='/update/:id' <UpdateUsers /> Dynamic

Dynamic Routes with Parameters

<Route path='/update/:id' element={<UpdateUsers />} />

// In UpdateUser.jsx:
import { useParams } from 'react-router-dom';

function UpdateUser() {
  const {id} = useParams();
  console.log(id);  // "abc123" if URL is /update/abc123
}

How It Works:

  1. User clicks: <Link to="/update/abc123">Edit</Link>
  2. URL changes to /update/abc123
  3. React Router matches /update/:id
  4. Renders UpdateUser component
  5. Component gets id = "abc123" from useParams
  6. Uses ID to fetch user data

Navigation Methods

Method 1: Link Component

import { Link } from 'react-router-dom';

<Link to="/create">Add User</Link>
<Link to={`/update/${user._id}`}>Edit</Link>

// Like <a> tag but without page reload

Method 2: useNavigate Hook

  const Submit = (e) => {
    e.preventDefault();
    axios.post(url, data)

  }

üéØ When to Use Each:
  • Link: For clickable navigation (buttons, menu items)
  • useNavigate: For programmatic navigation (after form submit, authentication)

Complete Navigation Flow

Example: Updating a User

  1. User on home (/): Sees table of users
  2. Clicks "Update" button: <Link to={`/update/${user._id}`}>
  3. URL changes: /update/abc123
  4. Router matches: /update/:id route
  5. UpdateUser renders: Gets id from URL
  6. Fetches data: axios.get('/getuser/abc123')
  7. Pre-fills form: With fetched data
  8. User edits & submits: Form handler runs
  9. Updates database: axios.put('/updateuser/abc123', data)
  10. Redirects home: navigate('/')
  11. Shows updated list: useEffect fetches new data

üì° Axios - HTTP Requests

üí° What is Axios?
A promise-based HTTP client for making API requests to your backend server.

Why Axios?

  • ‚úÖ Simpler syntax than fetch()
  • ‚úÖ Automatically transforms JSON
  • ‚úÖ Better error handling
  • ‚úÖ Request/response interceptors

Installation

npm install axios

Import in Component

import axios from 'axios';

HTTP Methods in Your App

1. GET - Fetch Data

Get All Users:

// In Users.jsx
useEffect(() => {
  axios.get("http://localhost:3001")
    .then(result => {
      console.log(result.data);  // Array of users
      setUsers(result.data);
    })
    .catch(err => console.log(err));
}, []);

What happens:

  • Sends GET request to server
  • Server responds with JSON array of users
  • result.data contains the array
  • Updates state with data

Get Single User:

// In UpdateUser.jsx
axios.get('http://localhost:3001/getuser/' + id)
  .then(result => {
    setName(result.data.name);
    setEmail(result.data.email);
    setAge(result.data.age);
  })
  .catch(err => console.log(err));

2. POST - Create New Data

// In CreateUser.jsx
const Submit = (e) => {
  e.preventDefault();
  
  axios.post("http://localhost:3001/createuser", {
    name: name,
    email: email,
    age: age
  })
  .then(result => {
    console.log(result);
    navigate('/');  // Redirect after success
  })
  .catch(err => console.log(err));
}

Breakdown:

  • axios.post(url, data) - sends POST request
  • Second parameter is the data object
  • Server receives data in req.body
  • Returns created user object

3. PUT - Update Existing Data

// In UpdateUser.jsx
const Update = (e) => {
  e.preventDefault();
  
  axios.put("http://localhost:3001/updateuser/" + id, {
    name: name,
    email: email,
    age: age
  })
  .then(result => {
    console.log(result);
    navigate('/');
  })
  .catch(err => console.log(err));
}

Difference from POST:

  • POST creates new resource
  • PUT updates existing resource
  • PUT requires ID in URL

4. DELETE - Remove Data

// In Users.jsx
const handleDelete = (id) => {
  axios.delete(`http://localhost:3001/deleteuser/${id}`)
    .then(res => {
      console.log(res);
      // Update UI without reloading
      setUsers(users.filter(user => user._id !== id));
    })
    .catch(err => console.log(err));
}

Smart UI Update:

  • Instead of reloading page
  • Filter out deleted user from state
  • React automatically updates table

Promise Chain Explained

axios.get(url)
  .then(result => {
    // Success! result.data has the response
    console.log(result.data);
  })
  .catch(err => {
    // Error! Network issue or server error
    console.log(err);
  });

Axios Request/Response Structure

Property Description Example
result.data Response body [{name: "John", ...}]
result.status HTTP status code 200, 404, 500
result.headers Response headers {"content-type": "application/json"}
result.config Request configuration {url, method, headers}
‚ö†Ô∏è Common Mistakes:
  • Wrong URL: Check port number (3001) and endpoint spelling
  • CORS errors: Make sure server has app.use(cors())
  • Forgetting .data: Response is in result.data, not result
  • Not handling errors: Always add .catch()

üéì Key React Concepts Summary

1. Components

  • Building blocks of UI
  • JavaScript functions that return JSX
  • Can be reused throughout app
  • Export with export default

2. JSX

  • JavaScript + XML (HTML-like syntax)
  • Use className instead of class
  • Use {} for JavaScript expressions
  • Self-closing tags: <input />
// Valid JSX:
<div className="container">
  <h1>{user.name}</h1>
  <p>{2 + 2}</p>
</div>

3. State

  • Data that changes over time
  • Triggers re-render when updated
  • Created with useState()
  • Each component has its own state
const [count, setCount] = useState(0);
setCount(count + 1);  // Updates and re-renders

4. Props

  • Pass data from parent to child
  • Read-only (immutable)
  • Like function parameters
<UserCard name="John" age={25} />

function UserCard({ name, age }) {
  return <div>{name} - {age}</div>
}

5. Hooks

  • Add React features to functional components
  • Must start with "use"
  • Only call at top level
Hook Purpose
useState Manage state
useEffect Side effects (API calls)
useParams Get URL parameters
useNavigate Programmatic navigation

6. Data Flow

Unidirectional Data Flow:
  • Data flows parent ‚Üí child (via props)
  • Child can't directly modify parent's state
  • Child notifies parent via callback functions

Question 1: What happens when you click "Delete"?

Answer:

  3. Server removes user from MongoDB

Answer:

  • Runs after component renders (avoids blocking)
  • Prevents infinite loops
  • Can control when it runs with dependencies
  • Separates data fetching from rendering logic
